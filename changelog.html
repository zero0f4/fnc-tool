v1.4 2026-04-30 Huidig

JSCU Logging — schone filteropzet

  • FEAT Twee onafhankelijke filters: categorie-chip (klik = drill in één categorie) en vrije zoektekst (zoekt door Event ID, categorie, subcategorie, beschrijving, channel, provider en GPO-pad).
  • FEAT Bij typen wordt een actieve categorie-chip automatisch losgelaten — je zoekt dan door alle 194 events.
  • FEAT Status-banner "Actieve filters" met × per filter om individueel te wissen.
  • FIX Filterresultaat werd niet correct toegepast op categoriechips; haystack-filter herschreven als één gecombineerde lowercase-string voor robuuste deelmatch.
  • FIX Self-heal voor stale service-worker — bij missende JSCU_EVENTS-data wordt SW automatisch ge-unregistreerd en de pagina herladen.
v1.3 2026-04-30 Eerder

Toegevoegd

  • FEAT JSCU Werkblad — per Event ID een status (Te beoordelen / Actief / Niet actief / N.v.t.) en notitie, met statistiekenbalk, Export/Import JSON en Reset.
  • FEAT JSCU Logging Index — categorie-chips bovenaan met aantal Event IDs per categorie; klikbaar als filter.
  • FEAT ABRO Werkblad — Import — eerder geëxporteerd CSV/XLSX-werkblad weer inlezen (status + notitie per eis-ID).
  • FEAT Cloud → TBB 4 lock in ABRO Werkblad: bij type toepassing Clouddienst wordt TBB-niveau automatisch op 4 gezet en zijn andere niveaus uitgeschakeld.
  • DOC MITRE D3FEND-sectie toegevoegd aan Kennisbank — 6 defensieve tactieken (Model, Harden, Detect, Isolate, Deceive, Evict) en mapping naar ABRO & JSCU.
  • DOC Wijzigingsblad (deze pagina) — overzicht van versies en wijzigingen.
  • FEAT Upload-icoon toegevoegd aan iconenset (Lucide-stijl) voor importknoppen.
  • DOC Disclaimer aangescherpt — expliciete melding dat de tool fouten of verkeerde interpretaties kan bevatten en dat de gebruiker altijd zelf verantwoordelijk is voor controle, toepassing en advies. Geen aansprakelijkheid. Korte disclaimer-banner ook op ABRO Werkblad en JSCU Logging.
  • FIX Service-worker network-first voor HTML/JS/CSS — updates komen voortaan automatisch door zonder cache-clear (sw bumped naar v10).
  • FIX TBB-vaststelling correct toegewezen aan opdrachtgever, niet aan NBIV. Tekst op TBB Stelsel-pagina (typen toepassingen + samenvattingstabel) en home-disclaimer aangepast: de aanbestedende dienst / Rijksoverheid bepaalt het TBB-niveau en het type toepassing; het NBIV adviseert en accrediteert.

Aangepast

  • DATA Cloud-eisen alleen op TBB 4: regels 5.2.4, 5.2.5, 5.3.2 en 5.3.4 zijn van tbb:[1,2,3,4] teruggebracht naar tbb:[4]. Verbodsregel 5.1.1 blijft [1,2,3].
  • DATA TBB Stelsel — H5 Cloud-rij: alleen ✓ bij TBB‑4; TBB‑3, TBB‑2 en TBB‑1 nu ✗. Toelichting onder de tabel herschreven.
  • DATA Typen toepassingen — Clouddienst: groene callout "Cloud kan uitsluitend op TBB‑4" toegevoegd; tegenstrijdige TBB-1/2-uitzonderingen verwijderd.
  • DOC Handleiding uitgebreid: §05a (Werkblad cloud-lock + import), §08 (D3FEND in Kennisbank), §09 (JSCU Werkblad), FAQ.
  • DOC Bronnen: MITRE D3FEND-bron toegevoegd naast ATT&CK.
  • DOC Home-tegel "Kennisbank" beschrijving aangepast: ATT&CK + D3FEND.

Opgelost

  • FIX JSCU-pagina laadde geen events — verkeerd databestand werd opgehaald (abro-data.js) en er werd verwezen naar een verwijderd catNav-element. Beide gerepareerd; tabel en zoeken werken nu.
  • FIX Service-worker cache opgehoogd naar fnc-tool-v9; jscu-events.js en changelog.html nu in de pre-cache.
